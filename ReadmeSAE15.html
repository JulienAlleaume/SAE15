SAE 15 scraper des données

Par manque de temps et de compréhension nous n’avons pas eu le temps de faire les graphiques GnuPlot.

Notre programme se compose en trois codes, un code qui scrape les données parkings des voitures, un deuxième pour les parkings à vélos et enfin un dernier qui crée des fichiers lisible par GnuPlot(Le troisième code n'est pas fini, nous avons rencontré beaucoup de problèmes)

Les deux premiers codes n'utilise pas de fonction, nous n'avons pas pu les mettre en forme pour faire un programme plus ergonomique.

Pour commencé à récupérer des données il faut exécuter les deux programme voiture et vélo. Il vont récolté des données toute les 5 muinutes et les mettre dans un fichier ' .svg'.

  • Programme SAE15Voiture
    - En entrée les données du site web DataMontpellier. - En sortie plusieurs fichiers:

  • Un fichiers avec les données brutes de chaque parkings pour chaque parkings.

  • Un avec toutes les données brutes dans un seul fichiers.

  • Un avec le calcul de la moyenne d eplace libre nommé 'DonnéesMoyenneLibre'.

  • Un avec le calcul de l'ecart type des place libre nommé 'DonnéesEcratType.log'.
  • Un avec le calcul pourcentage de place libre par rapport au total nommé 'DonnéesPourcentages'.
  • Un avec des données importante et exploitable par le code 'SAE15Gnu' nommé 'DATAVoiture.svg'.

  • Programme SAE15Velo

    - En entrée les données du site web DataMontpellier.
    - En sortie plusieurs fichiers:

  • Un fichiers avec les données brutes du parking a velo en xml nommé 'velo.xml'.

  • Un avec des données importante et exploitable par le code 'SAE15Gnu' nommé 'DATAVelo.svg'.
